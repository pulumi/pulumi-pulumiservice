### Generation tooling fix (codegen, not hand-edit)
- `provider/tools/scaffold-metadata/main.go` rule (5): case-insensitive
  path-param ↔ response-body alias. When OpenAPI is inconsistent (path
  uses `{changeRequestID}`, response returns `changeRequestId`), the
  scaffolder now emits a normalizing rename so the same logical field
  surfaces under one Pulumi-side name.
- Same file: `idFormat` is now always-derived (from path-param order with
  renames applied), not preserve-existing. No hand-curated idFormats
  exist today, and a stale idFormat after a rename change silently
  breaks ID synthesis.
- Re-running `go generate ./provider/pkg/cloud/...` now auto-derives
  `renames: {changeRequestId: changeRequestID}` and updates
  `idFormat` for `EnvironmentDraft` + `OpenEnvironmentRequest` —
  no hand edits to `metadata.json`.

### Example bug fixes (collateral, found by Phase 1 baseline)
Examples were referencing path-param INPUTS as if they were state OUTPUTS.
The codebase enforces input/output disjointness (`TestPathParamsAreInputOnly`
in rest/schema_test.go); fixed examples to use config values for chained
path-params and to reference legitimate cloud-side outputs:

- `examples/v2/environments/{yaml,typescript,python,go,csharp,java}` — `${environment.name}` → `${environment.id}` in stack output (path-params not surfaced)
- `examples/v2/default-organization/{yaml,typescript,python,go,csharp,java}` — `${default.orgName}` → `${serviceOrg}` (config); also export `default.GitHubLogin` (real output)
- `examples/v2/environment-config/{yaml,typescript,python,go,csharp,java}` — `${draft.changeRequestID}` → `${draft.changeRequestId}` (auto-derived rename now matches response casing)
- `examples/v2/insights-account/{yaml,typescript,python,go,csharp,java}` — `${account.accountName}` → reuse the local config-derived value via dependsOn; export `account.name` (output) instead
